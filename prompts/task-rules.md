# Task Generation Rules

1. Generate complete, functional implementation code. No TODO comments, no placeholder logic, no stub functions.
2. Follow existing patterns in the codebase. Match naming conventions, file organization, and architectural patterns you observe.
3. Only generate code related to this work package. Do not refactor or modify unrelated files.
4. Place all generated files in the correct layer directory.
5. Only import existing dependencies. If a new dependency is absolutely required, note it clearly in a comment.
6. Write production-quality code with proper error handling, input validation, and edge case coverage.
7. If tests are part of the project convention (check CLAUDE.md), generate corresponding test files.
