# Feature Generation Rules
> Inherits: _base.md

1. This is a FEATURE — implement the complete feature as described.
2. Follow the existing architectural patterns for similar features in the codebase.
3. If the feature requires API endpoints, follow the existing endpoint naming and response format conventions.
4. If the feature has a UI component, follow the existing component structure and styling patterns.

## Using Clarification Context

- **Acceptance criteria** → these are your primary success conditions — implement all of them
- **Data model details** → use the exact fields, types, and constraints specified
- **API contract** → implement the exact method, path, and payload shape
- **UI behavior** → match the described interaction flow and states
- **Auth/permissions** → implement access control as specified
