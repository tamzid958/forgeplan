# Help — Show Available Commands

Display all forgeplan commands with descriptions and examples. No configuration loading needed.

## Output

Print the following:

```
forgeplan — Forge Code from Plans
═══════════════════════════════════════════════════════════════════════

  COMMANDS
  ────────────────────────────────────────────────────────────────────

  wp <ID> [--dry-run]     Process a single work package
                          Creates branch, generates code, runs tests,
                          commits, pushes, creates PR, updates OpenProject.
                          --dry-run: preview routing and branch without
                          side effects.

  list [--sprint <name>]  List work packages for current sprint
                          Shows assigned-to-you WPs first. Displays
                          priority, type, status, assignee, and layer.
                          Default: top 50 by priority.

  list <ID>               Show WP details with hierarchy
                          Displays parent, siblings, children, and
                          relations for a specific work package.

  batch <ID,ID,...>       Process multiple WPs sequentially
                          Runs the full wp pipeline for each ID.
                          Prints a summary at the end.

  queue                   Auto-discover and process ready WPs
                          Finds WPs with pickup_status. Shows your
                          assigned WPs first, then unassigned.
                          Lets you pick which to process.

  breakdown <EPIC_ID>     Decompose an Epic into work items
  [--dry-run]             Analyzes the Epic description and breaks it
                          into Tasks, SubTasks, and Bugs. Presents
                          the breakdown for review, then creates
                          child WPs in OpenProject on approval.
                          --dry-run: preview breakdown without creating.

  init                    Interactive project setup
                          Configures OpenProject connection, detects
                          layers, toolchain, hooks, and status mappings.
                          Creates forgeplan.config.json + forgeplan.local.json.

  rollback <ID>           Undo a previous generation
                          Closes PR(s), deletes branch(es), reverts
                          WP status, posts rollback comment.
                          Handles multi-layer rollback automatically.

  doctor                  Health check diagnostic
                          Validates config, toolchain, OpenProject
                          connectivity, repos, hooks, and git auth.

  help                    Show this help message

  CONFIG FILES
  ────────────────────────────────────────────────────────────────────

  .env                    Secrets only (OP_API_KEY). Gitignored.
  forgeplan.config.json   Shared project config. Committed to git.
  forgeplan.local.json    Machine-specific config. Gitignored.
                          Generated by 'init'. Contains toolPaths,
                          hookConventions, and layerOverrides.

  PIPELINE (what 'wp' does)
  ────────────────────────────────────────────────────────────────────

   1  Load config          shared + local + .env
   2  Resolve tools        dotnet, node, gh, etc.
   3  Fetch WP + context   hierarchy, siblings, relations, comments
   4  Route to layer(s)    category → map → fallback heuristics
   5  Quality gate          reject empty descriptions
   6  Claim + status       assign to you, set IN PROGRESS
   7  Derive branch        type/WP-{id}-{slug} from hook conventions
   8  Per-layer loop:
      8a  Branch            create or resume existing
      8b  Generate code     + tests if parity required
      8c  Lint/format fix   auto-fix + pre-commit dry run
      8d  Test suite        full run + transitive fix
      8e  Build             validate with buildCmd
      8f  Commit            sanitized message + trailer
      8g  Push              to remote
      8h  Create PR         with --base explicit
   9  Cross-link PRs       related PRs section in each PR
  10  Update OpenProject   status + summary comment
  11  Log summary          logs/run-summary.jsonl

  RESULT TYPES
  ────────────────────────────────────────────────────────────────────

  SUCCESS    Code generated, all checks passed → PR created
  PARTIAL    Code generated, some checks failed → [WIP] PR created
  FAILURE    No code generated → no commit

  EXAMPLES
  ────────────────────────────────────────────────────────────────────

  /forgeplan wp 16549              # process WP #16549
  /forgeplan wp 16549 --dry-run    # preview only
  /forgeplan list                  # show current sprint WPs
  /forgeplan list 16549            # show WP #16549 with hierarchy
  /forgeplan list --sprint "S-12"  # show WPs for sprint S-12
  /forgeplan batch 100,101,102     # process three WPs
  /forgeplan queue                 # auto-discover ready WPs
  /forgeplan breakdown 16461       # break Epic into Tasks/SubTasks/Bugs
  /forgeplan breakdown 16461 --dry-run  # preview breakdown only
  /forgeplan rollback 16549        # undo generation
  /forgeplan init                  # set up project
  /forgeplan doctor                # health check
```
